Mobile Family Bill Splitter Automation

Turning a monthly 30-minute chore into a 30-second email forward

Link to Github Repo

What I Built

I got tired of manually parsing our 10-person family plan bill every month (mostly friends sharing costs), so I built a fully automated system that does it for me. Now, the moment I get the bill notification on my phone, I can generate the cost breakdown in seconds without ever leaving the file viewer. It turns a manual 15-minute spreadsheet chore into a single tap. Every month I'd spend 15 minutes going through T-Mobile's 6-page PDF bill, manually extracting each person's charges, calculating splits, and sending everyone their portion. It was frustrating why T-Mobile would not automatically do it in the bill when Verizon and AT&T do, but now I don't mind since I learned a lot during this build. It wasn't a huge time sink honestly, but the motivation to do it was basically zero. Classic "I'll spend 10 hours automating a 15-minute task" situation.

How It Works

On-Device Python Environment: Instead of relying on external servers, I cloned my repository directly onto my iPhone using a-shell, a terminal emulator that allows Python scripts to run locally on iOS.
iOS Shortcut Integration: I configured a custom Apple Shortcut that acts as a bridge between the iOS share sheet and the Python script.
Smart PDF Parsing: This was the trickiest part. I tried OCR and ML approaches first, but T-Mobile's PDFs are messy. Ended up using keyword-based text extraction to grab exactly what I needed from the bill summary and do simple maths to get exactly how much each person owes.

Technical Challenges

The biggest challenge was optimizing the workflow for a mobile environment. I had to ensure the Python environment within a-shell was robust enough to handle the data processing libraries and configure the input arguments so the script could dynamically accept the PDF path passed down from the iOS Shortcut.

Impact

Been running rock-solid since September 2024. Added more features over time and made it generic so other families can adapt it. Saves me 15 minutes monthly, but more importantly, I never have to think about bill splitting again.

Technologies Used: Python, a-shell, pandas, PDF parsing, YAML configs
